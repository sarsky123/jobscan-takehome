# Jobscan Take-Home: Resume-to-Job Recommendation — Spec

## Overview

Minimal, embedding-based resume-to-job recommendation system. Small corpus (200–2000 jobs). **Simplest workable system** over complex infrastructure.

---

## Tech Stack

| Layer   | Choices |
|--------|---------|
| Backend | Python 3.11+, FastAPI, Uvicorn |
| AI/ML   | OpenAI `text-embedding-3-small`, NumPy, FAISS (`faiss-cpu`) |
| Frontend | Vanilla HTML/CSS/JS only (no frameworks) |

---

## Principles

- **Pragmatism:** No DB, message queues, or background workers. In-memory + local files (`.bin`, `.json`).
- **Correctness:** Cosine similarity via FAISS `IndexFlatIP`; L2-normalize all vectors before index and search.
- **Robustness:** Truncate long inputs; retry with backoff for OpenAI rate limits (429).
- **Clean code:** Pydantic validation, type hints, clear names; comment only non-obvious logic.

---

## Intended Implementation

### 1. Offline ingestion service — **Implemented**

Isolated service in **`ingestion/`** so it can be scaled independently (own deps, env-configured paths, run as one-off or in a separate container/process).

| Item | Status | Notes |
|------|--------|------|
| **Service** | Done | `ingestion/`: config, loader, embedding, index_builder, run |
| **Input** | Done | All `*.json` in `JOBS_INPUT_DIR` (default `storage/feed/`); each file = JSON array of job objects |
| **Job schema** | Done | Uses `title`, `organization` (or `company`), `description_text` (or `description`) |
| **Text for embedding** | Done | `title + "\n\n" + organization + "\n\n" + description_text`; truncate at 30k chars |
| **Embeddings** | Done | OpenAI client, `text-embedding-3-small`, batches of 100 |
| **Retries** | Done | Tenacity exponential backoff on `RateLimitError` (429), 5 attempts |
| **L2 normalization** | Done | NumPy; divide by per-row L2 norm (zeros → norm 1) |
| **FAISS** | Done | `faiss.IndexFlatIP(1536)`, add normalized vectors |
| **Persistence** | Done | Scaling-ready storage split: vectors + documents (see `storage/` layout below) |

**Env (optional):** `JOBS_INPUT_DIR`, `FAISS_INDEX_PATH`, `JOB_IDS_PATH`, `DOCUMENTS_PATH` — override defaults to point at different volumes when scaling.

**Run:** From repo root, `OPENAI_API_KEY` required. `python -m ingestion`. For a dedicated ingestion env: `pip install -r ingestion/requirements.txt` then `python -m ingestion`.

## Data & File Layout

```
storage/                # Storage layer (mimics future S3 + VectorDB + DocumentDB)
  feed/                 # Raw job feeds (future: S3)
    *.json              # Seed job files (each = array of job objects)
  vectors/              # Vector store (future: Pinecone/Milvus)
    faiss_index.bin     # Generated by ingestion (local stand-in for vector DB)
    job_ids.json        # Generated by ingestion (FAISS idx -> job_id)
  documents/            # Document store (future: DynamoDB/MongoDB)
    jobs.json           # Generated by ingestion ({job_id: job_object})

ingestion/              # Isolated ingestion service (scalable independently)
  __init__.py
  __main__.py            # Entrypoint: python -m ingestion
  config.py              # Env: JOBS_INPUT_DIR, FAISS_INDEX_PATH, JOB_IDS_PATH, DOCUMENTS_PATH
  loader.py              # Discover & load job JSONs, text_to_embed
  embedding.py           # OpenAI embed_batch + retry, L2 normalize
  index_builder.py       # FAISS build & persist
  run.py                 # Orchestration
  requirements.txt       # openai, numpy, faiss-cpu, tenacity only

backend/
  requirements.txt       # API deps (fastapi, uvicorn, …)

SPEC.md                  # This file
```

## Run Commands

```bash
# ---- Option A: single venv (repo root) ----
python3 -m venv .venv
.venv/bin/pip install -r backend/requirements.txt   # API + shared deps
# Ingestion (requires OPENAI_API_KEY):
export OPENAI_API_KEY='your-key'
.venv/bin/python -m ingestion

# ---- Option B: ingestion as isolated service (own venv) ----
python3 -m venv .venv-ingestion
.venv-ingestion/bin/pip install -r ingestion/requirements.txt
export OPENAI_API_KEY='your-key'
# Optional: point at different paths (e.g. mounted volumes)
# export JOBS_INPUT_DIR=/data/feed
# export FAISS_INDEX_PATH=/data/vectors/faiss_index.bin
# export JOB_IDS_PATH=/data/vectors/job_ids.json
# export DOCUMENTS_PATH=/data/documents/jobs.json
.venv-ingestion/bin/python -m ingestion

# ---- API (when implemented) ----
.venv/bin/uvicorn backend.main:app --reload

# Frontend: static files served by backend or open frontend/index.html
```
